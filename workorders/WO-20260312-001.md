---
title: OAuth Admin Bootstrap for Local/Dev Login
id: WO-20260312-001
owner: platform
created: 2026-03-12

summary:
  Add a safe OAuth admin bootstrap path for local/dev so approved emails can sign in as admins without manual database updates after first login.

scope:
  - Add allowlist-based admin role elevation for OAuth users in the backend auth flow.
  - Align GitHub OAuth redirect handling with the existing Google redirect URI configuration pattern.
  - Document required OAuth and admin bootstrap environment variables in `.env.example`.
  - Update `CHANGELOG.md` after the work is completed.

files to be added/changed:
  - workorders/WO-20260312-001.md
  - src/routes/auth.py
  - .env.example
  - CHANGELOG.md

acceptance_criteria:
  - A user whose email appears in the configured OAuth admin allowlist receives the `admin` role on first OAuth login.
  - An existing OAuth user in the allowlist is promoted to `admin` on a subsequent OAuth login without manual SQL updates.
  - Google and GitHub OAuth both honor configured redirect URIs.
  - `.env.example` documents the OAuth client settings and the admin allowlist variable without committing secrets.

notes:
  - This work order intentionally avoids editing `docker-compose.yml` because it is outside the allowed edit paths in `AGENTS.md`.
  - Compose users must still pass the documented OAuth environment variables into the `web` container at runtime.
